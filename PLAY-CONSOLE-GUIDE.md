# Play Console 제출 체크리스트 — 지하철 화장실 찾기

이 문서를 **핸드폰이나 다른 창에 띄워놓고** Play Console 화면 보면서 한 단계씩 따라가세요.
각 단계는 클릭 순서 그대로 적혀있습니다.

---

## 준비: 이 값들을 미리 복사해둡니다

```
개인정보처리방침 URL: https://subwaytoilet.vercel.app/privacy.html
개발자 이메일:       maspiece83@gmail.com
```

---

## Step 1. 앱 콘텐츠 페이지 진입

1. https://play.google.com/console 접속
2. 왼쪽 앱 목록에서 **지하철 화장실 찾기** 선택
3. 왼쪽 메뉴에서 **정책 및 프로그램** 클릭 → 하위 메뉴 **앱 콘텐츠** 클릭

→ 여러 항목이 목록으로 보입니다. 아래부터 하나씩 처리합니다.

---

## Step 2. 개인정보처리방침 (제일 쉬움)

1. **개인정보처리방침** 카드 → **시작** 또는 **관리** 버튼
2. URL 입력칸에 아래 값 붙여넣기:

```
https://subwaytoilet.vercel.app/privacy.html
```

3. **저장** 버튼
4. 뒤로 가서 **앱 콘텐츠** 목록으로 복귀

체크: [ ] 완료

---

## Step 3. 광고 (Ads)

1. **광고** 카드 → **시작/관리**
2. 질문: "앱에 광고가 포함되어 있나요?"
3. 답: **예, 앱에 광고가 있습니다** 선택
4. **저장**

체크: [ ] 완료

이유: AdSense가 있으니까 무조건 "예"입니다.

---

## Step 4. 앱 액세스 권한 (App access)

1. **앱 액세스 권한** 카드 → **시작/관리**
2. 질문: "로그인 또는 특정 지역 등 제한된 기능이 있나요?"
3. 답: **모든 기능은 특별한 액세스 권한 없이 사용할 수 있습니다** 선택
4. **저장**

체크: [ ] 완료

이유: 본 앱은 로그인이 없고 모든 사용자가 모든 기능 사용 가능.

---

## Step 5. 콘텐츠 등급 (Content rating)

1. **콘텐츠 등급** 카드 → **시작**
2. 이메일 입력: `maspiece83@gmail.com` 자동으로 채워져 있을 수 있음. 아니면 직접 입력.
3. **카테고리 선택** 화면에서 **참조(Reference)** 또는 **유틸리티** 선택
   - "참조, 뉴스, 교육" 카테고리가 있으면 그걸 선택
4. 설문 답변:
   - 폭력: **아니요**
   - 성적인 콘텐츠: **아니요**
   - 욕설: **아니요**
   - 통제물질(약물, 도박): **아니요**
   - 사용자 생성 콘텐츠: **아니요**
   - 이용자 간 소통/메시지: **아니요**
   - 사용자의 정확한 위치 공유: **아니요**
   - 웹 브라우징 기능: **아니요**
   - 디지털 구매: **아니요**
5. **저장** → **설문 제출**

체크: [ ] 완료

이유: 지하철 화장실 위치 안내만 하는 단순 유틸리티. 전부 "아니요".

---

## Step 6. 타겟층 및 콘텐츠 (Target audience)

1. **타겟층 및 콘텐츠** 카드 → **시작/관리**
2. **타겟 연령층** 선택:
   - 만 13세 이상 / 만 14세 이상 / 만 17세 이상 / 만 18세 이상 중 **"만 18세 이상"** 선택 (또는 "만 13세 이상")
   - ※ 아동 정책을 피하려면 최소 만 13세 이상을 권장
3. "아동의 관심을 끌 수 있나요?" → **아니요**
4. **다음**
5. 광고 관련 질문이 나오면:
   - "앱에 광고가 포함되어 있습니까?" → **예**
   - "아동에게 부적절한 광고가 차단되도록 보장할 수 있습니까?" → **예** (AdSense가 연령 필터링함)
6. **저장**

체크: [ ] 완료

---

## Step 7. 뉴스 앱 / COVID-19 / 정부 앱 / 금융 상품

이 네 개는 전부 **아니요 / 해당 없음**입니다.

- **뉴스 앱** → "뉴스 앱이 아님" 선택 → 저장
- **COVID-19 접촉자 추적 앱** → "해당 없음" → 저장
- **정부 앱** → "정부 앱이 아님" → 저장
- **재정 상품 및 서비스** → "해당 없음" → 저장

체크: [ ] 모두 완료

---

## Step 8. 데이터 보안 (Data Safety) — 가장 복잡한 섹션

마음 단단히 먹고 천천히. **한 화면씩** 따라갑니다.

### 8-1. 진입

1. **데이터 보안** 카드 → **시작** 또는 **관리**
2. "개요" 화면이 뜸 → **다음** 클릭

### 8-2. 데이터 수집 및 보안 개요

질문: "앱이 필요한 사용자 데이터 유형을 수집하거나 공유합니까?"

답: **예** 선택

이유: AdSense가 광고 ID(Android Advertising ID)를 수집하니까 "예".

**다음** 클릭

---

### 8-3. 데이터 유형 선택 화면

카테고리가 쭉 나옵니다. 아래 목록 그대로 따라가세요.

**대부분 "수집 안 함"으로 두고, 딱 하나만 체크합니다.**

| 카테고리 | 수집? | 비고 |
|---|---|---|
| 위치 (Location) | ❌ | 체크 해제 |
| 개인 정보 (Personal info) | ❌ | 체크 해제 |
| 금융 정보 (Financial info) | ❌ | 체크 해제 |
| 건강 및 피트니스 | ❌ | 체크 해제 |
| 메시지 (Messages) | ❌ | 체크 해제 |
| 사진 및 동영상 | ❌ | 체크 해제 |
| 오디오 파일 | ❌ | 체크 해제 |
| 파일 및 문서 | ❌ | 체크 해제 |
| 캘린더 | ❌ | 체크 해제 |
| 연락처 (Contacts) | ❌ | 체크 해제 |
| 앱 활동 (App activity) | ❌ | 체크 해제 |
| 웹 방문 기록 | ❌ | 체크 해제 |
| 앱 정보 및 성능 | ❌ | 체크 해제 |
| **기기 또는 기타 ID (Device or other IDs)** | **✅** | **이것만 체크** |

체크 끝나면 **다음** 클릭.

---

### 8-4. "기기 또는 기타 ID" 상세 질문

"기기 또는 기타 ID"를 체크했으니 상세 질문 페이지가 뜹니다. 아래처럼 답변:

**질문 1: 이 데이터 유형이 수집됩니까?**
→ **예**

**질문 2: 이 데이터 유형이 공유됩니까?**
→ **예**
(이유: Google에 공유됨. AdSense가 제3자 서비스이므로 "공유"로 분류)

**질문 3: 이 데이터는 임시적으로만 처리됩니까?**
→ **아니요**
("임시 처리"는 앱 종료 시 바로 삭제되는 경우. AdSense는 장기 보관하므로 "아니요")

**질문 4: 사용자가 이 데이터 수집을 선택하지 않을 수 있습니까?**
→ **아니요, 이 데이터 수집은 필수입니다**
(앱 사용에 광고가 기본이므로 "필수"로 분류하는 게 안전)

**질문 5: 이 데이터를 수집하는 목적은 무엇입니까?** (복수 선택 가능)
→ **광고 또는 마케팅** 만 체크
- ❌ 앱 기능
- ❌ 분석
- ❌ 개발자 커뮤니케이션
- ✅ 광고 또는 마케팅
- ❌ 사기 방지, 보안 및 규정 준수
- ❌ 계정 관리
- ❌ 개인 맞춤 설정

**저장** 클릭 후 데이터 유형 목록으로 복귀.

---

### 8-5. 보안 관행 (Security practices)

데이터 유형 목록에서 **다음** 클릭하면 "데이터 보안" 화면이 뜹니다.

**질문 1: 데이터가 전송 중 암호화됩니까?**
→ **예, 데이터는 전송 중 암호화됩니다**
(이유: 모든 통신이 HTTPS)

**질문 2: 사용자가 자신의 데이터 삭제를 요청할 수 있는 방법을 제공합니까?**
→ **아니요, 사용자는 데이터 삭제를 요청할 수 없습니다**
(이유: 본인이 수집하는 데이터가 없으므로 삭제 요청 대상이 없음. AdSense 데이터는 Google이 관리)

**저장** 클릭.

---

### 8-6. Google Play의 가족 정책 준수 (Families Policy)

질문: "앱이 Google Play 가족 정책을 준수합니까?"
→ **아니요** 선택

(이유: 본 앱은 아동 대상이 아닙니다. Step 6에서 만 13세 이상으로 설정했으므로 가족 정책 대상이 아님)

---

### 8-7. 독립적 보안 검증

질문: "앱이 독립적인 보안 검증을 받았습니까?"
→ **아니요** 선택

(이유: MASA(Mobile App Security Assessment) 같은 공식 검증 받은 적 없으므로)

---

### 8-8. 데이터 보안 섹션 최종 제출

1. **미리보기** 화면에서 입력한 내용 검토
2. 맞으면 **제출** 클릭

체크: [ ] 데이터 보안 섹션 완료

---

## Step 9. 모든 앱 콘텐츠 항목이 체크되었는지 확인

**앱 콘텐츠** 목록 페이지로 돌아가서 모든 카드에 **"완료"** 또는 **초록색 체크 표시**가 되어 있는지 확인:

- [ ] 개인정보처리방침
- [ ] 앱 액세스 권한
- [ ] 광고
- [ ] 콘텐츠 등급
- [ ] 타겟층 및 콘텐츠
- [ ] 뉴스 앱
- [ ] COVID-19 접촉자 추적 앱
- [ ] 데이터 보안
- [ ] 정부 앱
- [ ] 재정 상품 및 서비스

다 체크되어 있으면 **공개 테스트 트랙**으로 가셔도 됩니다.

---

## Step 10. 공개 테스트 시작

1. 왼쪽 메뉴: **테스트 및 출시** → **테스트** → **공개 테스트**
2. 우측 상단 **새 버전 만들기** (이미 만들어진 버전이 있으면 그거 사용)
3. **검토 시작** → 문제 없으면 **공개 테스트에 출시**
4. 제출 후 Google 검토 대기 (보통 몇 시간 ~ 1~2일)

**정리:**

- 전체 과정은 20~30분 정도 예상
